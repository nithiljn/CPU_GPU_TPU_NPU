CPU : The General Brain

What is it?

The Swiss Army Knife of Computing

The CPU is designed to handle any task ‚Äî running your OS, playing audio, running Excel, compiling code. It's the master of sequential, complex logic.

CPUs are built around a few very powerful cores (4‚Äì32 in consumer chips). Each core has massive caches, branch predictors, out-of-order execution units, and all kinds of logic to run any code as fast as possible.

GPU : The Parallel Powerhouse

Graphics Processing Unit ¬∑ Invented for gaming ¬∑ Accidentally became the backbone of AI

The Problem it Solved

Born from Gaming, Became the AI Revolution

In the 90s, rendering 3D graphics meant doing the same math on thousands of pixels simultaneously. The CPU was terrible at this ‚Äî it's built for one complex task at a time, not a million simple tasks at once.

So engineers built a chip that trades a few smart cores for thousands of tiny, simpler cores. And this design turned out to be perfect for something else entirely matrix math in neural networks.

üé® Analogy

GPU = An army of 10,000 workers. Each worker is simple and can only do basic math, but all 10,000 work simultaneously. When the job is "do the same thing to a million numbers" this army crushes the genius professor.

How it works internally

SIMD ‚Äî Same Instruction, Multiple Data

The key concept is SIMD (Single Instruction, Multiple Data). Instead of executing one instruction on one number, GPUs execute one instruction on thousands of numbers simultaneously.

Inside a GPU, there are:

‚Üí Streaming Multiprocessors (SMs) : groups of 128 cores each
‚Üí CUDA Cores (NVIDIA) or Shaders (AMD) thousands of simple ALUs
‚Üí Tensor Cores : special units that do entire matrix multiplications in one clock cycle
‚Üí HBM (High Bandwidth Memory) : stacked memory with enormous bandwidth (~3 TB/s in H100)

TPU : Google's AI Weapon

Tensor Processing Unit ¬∑ Created by Google in 2016 ¬∑ Purpose-built for neural network math

Why TPU Was Created

Even the GPU Wasn't Enough

By 2013, Google realized that if people used voice search for 3 minutes a day, they'd need to double their data center capacity just for AI inference with GPUs. GPUs were great, but still too general purpose and too power hungry.

So Google secretly built a chip that does only one thing: matrix multiplications for neural networks. No 3D rendering, no gaming just pure tensor math. The result was 15‚Äì30√ó more efficient than GPUs for inference tasks.

üè≠ Analogy

TPU = A specialized factory assembly line. Designed exclusively to build one product. Insanely fast and efficient at that one thing. Can't do anything else.

The Key Innovation

The Systolic Array : The Heart of a TPU

The magic inside a TPU is a systolic array ‚Äî a grid of simple multiply-accumulate (MAC) units arranged so data flows through them like water through pipes, doing matrix math without ever stopping.

SYSTOLIC ARRAY : 256√ó256 = 65,536 MAC UNITS IN TPU v1

Data flows horizontally ‚Üí, partial sums flow vertically ‚Üì. Every cell works every clock cycle. No wasted cycles.

This is different from a GPU where cores wait for data to be loaded. In a systolic array, data flows continuously like a conveyor belt so every single MAC unit is always working.

TPU v4 has 275 trillion operations per second (275 TOPS) using bfloat16 precision. Each TPU Pod links 4096 chips together via ultra-fast interconnects Google uses these to train Gemini.

TPU vs GPU

Key Differences from GPU

Less flexible: TPUs can't run arbitrary CUDA code. They're programmed via XLA (a compiler) and TensorFlow/JAX. You can't just "install CUDA" on a TPU.

More efficient: For the specific job of running transformer models, TPUs use significantly less power per operation.

Scale differently: TPU Pods connect thousands of chips with dedicated high speed optical interconnects, making them better for massive model training than GPU clusters connected by slower NVLink or InfiniBand.

NPU : AI on Your Device

Neural Processing Unit ¬∑ Embedded in phones & laptops ¬∑ Low power AI acceleration

Why NPU Exists

The Problem: You Can't Send Everything to the Cloud

Running AI on a GPU server means: your data leaves your device, there's network latency, it costs money per query, and it drains your battery waiting for responses.

For realtime tasks : face unlock, voice recognition, camera AI, live translation , you need AI that runs on device, instantly, using milliwatts of power. That's what an NPU does.

‚ö° Analogy

NPU = A tiny, super efficient specialist embedded in your phone. Doesn't need the internet. Runs constantly in the background using almost no battery.

How it works

Efficiency Over Raw Power

NPUs are designed for inference only (running already trained models), not training. The key design goals are completely different from GPUs/TPUs:

Power draw
~1W
GPU power draw
700W

NPUs achieve this through:

‚Üí INT8 / INT4 quantization : instead of 32-bit floats, they use tiny 4-bit or 8-bit integers. Less precision, but tiny models often don't need full precision for inference.

‚Üí Weight caching : model weights stay loaded in on-chip SRAM so there's no memory bandwidth waste.

‚Üí Dedicated datapath : the circuit is hardwired for specific operations. No control logic overhead like in GPUs.

Apple's Neural Engine (in M4 chip) can do 38 trillion operations per second while using only a fraction of the power of a GPU. It runs Face ID, Siri, camera processing, and on device LLMs like Apple Intelligence.

2012
